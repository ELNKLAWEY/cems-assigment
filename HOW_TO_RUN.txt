  COURSE ENROLLMENT MANAGEMENT SYSTEM (CEMS) - HOW TO RUN
═══════════════════════════════════════════════════════════════════

📋 QUICK COMMANDS
═══════════════════════════════════════════════════════════════════

1. TEST THE SYSTEM (Recommended First Step)
   ─────────────────────────────────────────
   dart run bin/test_cems.dart
   
   This will:
   • Run 28 automated tests
   • Verify all functionality works
   • Show you what features are available
   • Takes ~2 seconds

2. RUN WITH SAMPLE DATA (Best for Learning)
   ─────────────────────────────────────────
   dart run bin/cems_with_sample_data.dart
   
   This will:
   • Load 5 students, 6 courses, 16 enrollments
   • Let you explore all features immediately
   • Perfect for demos and learning
   • Recommended for first-time users

3. RUN EMPTY SYSTEM (Production Mode)
   ─────────────────────────────────────────
   dart run bin/cems_main.dart
   
   This will:
   • Start with empty database
   • You add your own students/courses
   • Best for real data entry
   • Production-ready
